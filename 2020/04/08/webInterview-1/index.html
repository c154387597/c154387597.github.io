严格模式与混杂模式

Doctype作用

Doctype：(Document Type)文档类型，<!DOCTYPE>声明叫做文件类型定义（DTD），它位于文档中最前面的位置，处于标签之前。如果你想制作符合标准的页面，一个必不可少的关键组成部分就是DOCTYPE的声明。确定了正确的Doctype，xhtml里面的标识和css才能正常生效(也就是说它会很负责地告诉浏览器怎么解释你的这个标签和你写的css)。既然它是一种声明，它的责任就是告诉浏览器文档使用哪种html或者xhtml规范。

  • Doctype可声明三种DTD类型，分别表示严格版本、过渡版本以及基于框架的 HTML 文档
  • 当浏览器厂商开始创建与标准兼容的浏览器时，他们希望确保向后兼容性。为了实现这一点，他们创建了两种呈现模式：严格模式和混杂模式
  • 严格模式（标准模式）：指浏览器按照W3C标准解析代码；浏览器以其支持的最高标准呈现页面
  • 混杂模式（兼容模式，怪异模式）：指浏览器用自己的方式解析代码；页面以一种比较宽松的向后兼容的方式显示

如何区分严格模式和混杂模式

  1. 如果XHTML、HTML 4.01文档包含形式完整的DOCTYPE，那么它一般以标准模式呈现。
  2. 包含过渡DTD和URI的DOCTYPE也导致页面以标准模式呈现，但是有过渡DTD而没有URI会导致页面以混杂模式呈现。
  3. DOCTYPE不存在或形式不正确会导致HTML和XHTML文档以混杂模式呈现。
  4. html5既然没有DTD，也就没有严格模式与宽松模式的区别，html5有相对宽松的语法，实现时，已经尽可能大的实现了向后兼容。
    “ html5不基于SGML（标准通用标记语言），因此不需要对DTD进行引用，但是需要Doctype来规范浏览器行为。”

严格模式与混杂模式的语句解析不同点

  1. 标准模式下，一个元素的高度是由其包含的内容来决定的，如果父元素没有设置高度，子元素设置一个百分比的高度无效。
  2. 标准模式下使用margin: 0 auto;可以使元素水平居中，但是在混杂模式下会失效；混杂模式下需要设置text-align:center;来进行水平居中。
  3. 混杂模式下盒子模型的高宽包括padding和border，而W3C标准中设置一个元素的高宽指的是content的高宽。
  4. 混杂模式下可以设置行内元素的高度，而标准模式下不生效。
  5. 混杂模式下设置图片padding会失效。
  6. 混杂模式下Table中的字体属性不能继承上层的设置。
  7. 混杂模式下white-space: pre会失效。

meta详解

  • <meta> 标签位于文档的头部，不包含任何内容。<meta> 标签的属性定义了与文档相关联的名称/值对。
  • <meta> 元素可提供有关页面的元信息（meta-information），比如针对搜索引擎和更新频度的描述和关键词。
  • <meta>由两个属性组成，分别是http-equiv属性和name属性

http-equiv属性

http的头文件作用，指示服务器在发送实际的文档之前先在要传送给浏览器的 MIME 文档头部包含名称/值对。<meta http-equiv="参数" content="具体的描述">

  1. X-UA-Compatible（浏览器采用何种版本渲染当前页面），一般都设置为最新模式，在各大框架中也很常见。

  2. cache-control（指定请求和相应遵循的缓存机制），指导浏览器如何缓存某个响应以及缓存多长时间。

  3. expires（网页到期时间），用于设定网页的到期时间，过期后网页必须到服务器上重新传输。

  4. refresh（自动刷新并指向某页面），网页将在设定的时间内自动刷新并调向设定的网址。

  5. Set-Cookie（cookie设定），如果网页到期，那么网页存在本地的cookie也会被删除。

  6. mobile-agent（开放手机适配）

    站点如果自行适配困难，可以在PC页面中做简单改造，百度协助实现适配效果。即：站长在站点PC页的源代码头部嵌入一行或多行Meta信息，由Meta信息来指明该PC页对应手机页的URL，以及该URL对应页面的格式，百度将根据用户终端类型选择最适合展示的页面

name属性

name属性主要用于描述网页，与之对应的是content，content是对name的具体描述，便于引擎抓取。<meta name="参数" content="具体的描述">

  1. keywords（关键字），用于告诉搜索引擎网站的关键字，在搜索引擎中输入keywords中写的内容，即可搜索到网页。

  2. description（网站内容描述），用于告诉引擎网站的主要内容，在根据关键字搜索出网页后，会显示description中的内容。

  3. viewport（移动端的窗口），这属于移动端页面会用到的属性，控制网页在移动端有较好的显示。

  4. robots（定义搜索引擎爬虫的索引方式），告诉爬虫哪些页面需要索引，哪些页面不需要索引。

  5. author（作者），用于标注网页作者

  6. generate（网页制作软件），用于标注网站的编码软件
  7. copyright（版权），用于注明网页的版权所有

  8. revisit-after（搜索引擎爬虫搜索时间），如果页面不是经常更新，为了减少搜索引擎爬虫对服务器带来的压力，可以设置一个爬虫访问时间。

  9. renderer（双核浏览器渲染方式），指定双核浏览器用哪种方式渲染。

HTML语义化

什么是语义化

根据内容的结构（内容语义化），选择合适的标签（代码语义化）便于开发者阅读和写出优雅的代码同时让浏览器的爬虫和机器很好地解析。

语义化优点

  • 易于用户阅读，样式丢失的时候能让页面呈现清晰的结构
  • 有利于seo，搜索引擎根据标签来确定上下文和各个关键字的权重
  • 方便其它设备解析，如盲人阅读器根据语义渲染网页
  • 有利于开发和维护，语义化更具可读性，代码更好维护，与css3关系更和谐

href和src的区别

请求资源不同

  1. href是Hypertext Reference的缩写，表示超文本引用。用来建立当前元素和文档之间的连接。
  2. 在请求src资源时会将其指向的资源下载并应用到文档中。

作用结果不同

  1. href用于当前文档和引用资源之间建立联系。
  2. src用于替换当前内容。

浏览器解析方式不同

  1. 若在文档中添加href，浏览器会识别该文档为css文件，并行下载资源并且不会停止对当前文档的处理。这也是为什么推荐使用link方式加载css，而不是@import方式。

    link是XHTML标签，除了可以加载css，还可定义RSS等其它事务；而@import属于css，只可以加载css。
    link引用css时，在页面载入同时加载；@import需要等页面加载完成后再加载。
    link是XHTML标签无兼容问题；@import是在css2.1提出，低版本浏览器不支持。
    link支持JavaScript控制DOM改变样式，而@import不支持。

  2. 当浏览器解析到src，会暂停其他资源的下载和处理，直到将该资源加载、编译、执行完毕，图片和框架等也如此，类似于将所指向资源应用到当前内容。这也是为什么建议把 js 脚本放在底部而不是头部的原因。

块级元素和行内元素

块级元素

  • 独占一行，不与其它元素并列
  • 可以容纳内联元素和其它块元素
  • 可控制宽高以及外边距和内边距，如果不设置宽度则默认为父级的100%
标签 描述
div 定义文档中的分区或节
p 定义段落
pre 定义预格式文本
h1-h6 定义 html 标题
ul 定义无序列表
ol 定义有序列表
li 定义列表的项目
hr 定义水平线
section 定义section
form 定义供用户输入的 html 表单
table 定义表格
tbody 定义表格中的主体内容
td 定义表格中的单元
th 定义表格中的表头单元格
thead 定义表格中的表头内容
tfoot 定义表格的页脚(脚注或表注)
。。。 。。。

行内元素

  • 与其它行内元素并列
  • 不能设置宽高，默认宽度为文字宽度
  • 内联元素只能容纳文本或其它内联元素
标签 描述
a 定义超链接
b 定义粗体字
i 定义斜体字
br 定义换行
button 定义按钮
img 定义图像
imput 定义输入控件
label 定义 input 元素的标注
q 标记一个短的引用
select 定义选择列表（下拉列表）
small 定义小号文本
span 组合文档中的行内元素
strong 定义强调文本
sub 定义下标文本
sup 定义上标文本
textarea 定义多行的文本输入控件
。。。 。。。

块级元素和行内元素的相互转换

可以通过display属性将块级元素(比如div)和行内元素进行相互转换

  • 使用display: inline设置对象做为行内元素显示
  • 使用display: block设置对象为块级元素显示
  • 使用display: inline-block设置对象为行内块元素。（CSS2.1 新增的值）

盒模型

页面渲染时，dom 元素所采用的布局模型。可通过box-sizing进行设置。

  • content-box (W3C 标准盒模型)： 盒模型的宽高等于 内容(content) 的宽高
  • border-box (IE 盒模型)： 盒模型的宽高等于 内容(content) + 填充(padding) + 边框(border) 的总宽高
  • padding-box： 盒模型的宽高等于 内容(content) + 填充(padding) 的总宽高
