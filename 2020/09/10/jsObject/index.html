面向对象

  • 一切事物皆对象
  • 对象具有封装、继承和多态特性
  • 对象与对象之间使用消息通信，各自存在信息隐藏

ECMAScript 是一门彻底的面向对象的编程语言，JavaScript 是其中的一个变种 (variant)。它提供了6种基本数据类型，即 Boolean、Number、String、Null、Undefined、Object。JavaScript 是通过原型（prototype）的方式来实现面向对象编程。

基于类的面向对象和基于原型的面向对象方式比较

在基于类的面向对象方式中，对象（object）依靠类（class）来产生。而在基于原型的面向对象方式中，对象（object）则是依靠构造器（constructor）利用原型（prototype）构造出来的。

在类式面向对象语言中，对象的状态 (state) 由对象实例 (instance) 所持有，对象的行为方法 (method) 则由声明该对象的类所持有，并且只有对象的结构和方法能够被继承；而在原型式面向对象语言中，对象的行为、状态都属于对象本身，并且能够一起被继承。

类式面向对象语言比如 Java，为了弥补无法使用面向过程语言中全局函数和变量的不便，允许在类中声明静态 (static) 属性和静态方法。在原型式面向对象中，除内建对象（build-in object）外，不允许全局对象、方法或属性的存在，也没有静态概念。所有语言元素（primitive）必须依赖对象存在。但由于函数式语言的特点，语言元素所依赖的对象是随着运行时（runtime）上下文（context）变化而变化，具体体现在指针this的变化。

使用函数构造器构造对象

ECMAScript允许通过构造器（constructor）创建对象。每个构造器实际上是一个函数（function）对象，该函数对象含有一个prototype属性用于实现基于原型的继承（prototype-based inheritance）和共享属性（shared properties）。

彻底理解原型链 (prototype chain)

在ECMAScript中，每个由构造器创建的对象都有一个指向构造器prototype属性值的隐式引用（implicit reference），这个引用称之为原型（prototype）。进一步，每个原型可以拥有指向自己原型的隐式引用（即该原型的原型），如此下去，就是所谓的原型链（prototype chain）。在具体的语言实现中，每个对象都有一个proto属性来实现对原型的隐式引用。

prototype是函数所独有的：由于函数也是对象，所以也拥有__proto__属性，prototype属性的作用在于该函数所实例化的对象可以找到公用的属性和方法
__proto__是对象所独有的：__proto__当访问一个对象的属性不存在时，就会去它的__proto__属性所指向的父级查找，直到__proto__属性为null

有了原型链，便可以定义属性隐藏机制，并通过这种机制实现继承。ECMAScript规定，当要给某个对象的属性赋值时，解释器会查找该对象原型链中第一个含有该属性的对象（原型本身就是一个对象，原型链即为一组对象的链。对象的原型链中第一个对象是对象本身）进行赋值。反之，要获取某个对象属性的值，解释器自然是返回该对象原型链中首先具有该属性的对象属性值。

object1 prototype1 prototype2
{ property1: value1,
property4: value4,
proto: prototype1 }		 { property2: value2,
property3: value3,
property4: value4,
proto: prototype2 }		 { property3: value3,
property5: value5,
proto: Object.prototype }

上表格中，object1 -> prototype1 -> prototype2 构成对象object1的原型链，根据上述属性隐藏机制，可以清楚看到prototype1对象中的property4属性和prototype2对象中的property3属性皆被隐藏。
